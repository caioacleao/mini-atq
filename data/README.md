# Reproducibility data for "Mini AT-Q"

This directory archives the **frozen-decode evaluation CSVs** (and run configs) behind
every number in the paper, so the manuscript's tables and figures can be regenerated
from a clean clone. The CSVs are the unit of record: each row is one frozen-decode
attempt (`solved`, `cost`/T-count, `num_moves`, action sequences) under
`controls=orbit, k=1`. The aggregated numbers live in the regenerated `outputs/*.json`.

## How to re-run
The analysis tools read `results_*/eval` at the **repo root**. To regenerate:

```bash
bash data/restore.sh        # copies data/results_* -> repo root
python3 tools/paper_analysis.py   # -> outputs/numbers.json (mechanism, Fisher p, gumbel-vs-muzero)
python3 tools/paper_generality.py # -> outputs/generality_numbers.json + Table V LaTeX rows
python3 tools/paper_hardening_numbers.py # -> outputs/hardening_numbers.json (delta-sweep, symlog generality, support boundary, mod_5_4 out-of-sample)
python3 tools/paper_mechanism_figure.py   # -> fig_mech_curves/strip + mechanism_table.json
python3 tools/paper_tail_statistic.py     # -> outputs/tail_statistic.json + fig_tail (needs the target tensors)
python3 tools/paper_grid_heatmap.py       # -> fig_grid_heatmap (compute grid)
# classical baseline (needs qiskit + pyzx): python3 tools/comparison/qiskit_compare.py
```

## Notes
- `barenco_tof_3` (mechanism) uses 5 train seeds; the generality targets use 3, except
  `gf_2pow3_mult` Huber which uses **6** (3 original + 3 matched-budget hardening seeds at the
  fair `max_num_moves=48`, 1500 steps). `solved` is constant across the 5 eval seeds per train
  seed; the train seed is the unit of replication.
- Hardening pass (2026-06-28): the delta-sweep (5 seeds), symlog generality (cuccaro/gf3), the
  categorical support-floor boundary (`[-80,0]`/`[-120,0]`), and the `mod_5_4` out-of-sample
  point are regenerated by `tools/paper_hardening_numbers.py` -> `outputs/hardening_numbers.json`.
- The published Best/AT-Q/gadget T-counts in Table I are external (Ruiz et al. 2025, Table 2),
  not derived from these CSVs. The PyZX column is reproducible via `tools/comparison/qiskit_compare.py`.
- The per-target heavy-tail statistic is computed from the target tensors (not these CSVs) by a
  numpy environment replica cross-validated against the JAX environment; see `tools/paper_tail_statistic.py`.
